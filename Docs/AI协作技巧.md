# AI 协作技巧

> 如何与 AI 高效协作开发游戏

---

## 🎯 核心原则

### 1. AI 是助手，不是替代者

**记住**：
- ✅ AI 能帮你写代码、找 bug、优化性能
- ❌ AI 不能替你做设计决策、审美判断、创意构思

**你的角色**：
- 游戏设计师：决定做什么
- 项目经理：规划怎么做
- 质量控制：检查做得好不好

**AI 的角色**：
- 代码生成器：快速实现功能
- Debug 助手：找出和修复问题
- 优化顾问：提升性能和代码质量

---

### 2. 给 AI 完整的上下文

**❌ 糟糕的提示词**：
```
帮我优化这个函数
[粘贴 50 行代码]
```

**问题**：
- AI 不知道这是什么项目
- AI 不知道这段代码的作用
- AI 不知道相关的其他代码
- AI 不知道优化的目标

**✅ 好的提示词**：
```
我在做一个 Bad North 风格的 3D 即时战略游戏。

项目结构：
- @Scripts/Config/GameConfig.cs - 游戏配置
- @Scripts/Systems/Combat/CombatSystem.cs - 战斗系统
- @Scripts/Entities/Units/Unit.cs - 单位类

当前问题：
弓箭手的攻击速度太慢，影响游戏体验。

相关代码：
1. 配置在 Config/UnitConfig.cs 第 45-60 行
2. 攻击逻辑在 Systems/Combat/CombatSystem.cs 第 120-150 行
3. 动画时长在 Entities/Units/Unit.cs 第 80-100 行

目标：
- 将攻击速度提高 30%（从 1.5s → 1.05s）
- 确保动画时长同步调整
- 验证不影响其他单位的平衡性
- 自动更新相关注释

请修改这 3 个文件，并说明改动的内容。
```

**优势**：
- AI 理解完整上下文
- AI 知道具体的目标
- AI 能协调多个文件的修改
- AI 会自动验证影响范围

## 💡 提示词工程技巧

### 技巧 1：使用 @ 引用文件

**语法**：
```
@文件路径 - 让 AI 读取文件内容
```

**示例**：
```
@Assets/Scripts/Config/UnitConfig.cs
@Assets/Scripts/Systems/Combat/CombatSystem.cs

请分析这两个文件，找出弓箭手攻击速度相关的代码
```

---

### 技巧 2：明确输出格式

**❌ 模糊的要求**：
```
帮我修复这个 bug
```

**✅ 明确的要求**：
```
请修复敌人 AI 卡住的问题。

要求：
1. 先分析问题原因（3-5 句话）
2. 给出解决方案（列出步骤）
3. 修改代码（完整代码块）
4. 说明修改的影响（可能的影响范围）
5. 建议测试方法（如何验证修复）
```

---

### 技巧 3：分步骤处理复杂任务

**示例：添加新单位**

**步骤 1：设计阶段**
```
我想添加一个"维京狂战士"单位，帮我设计：

属性要求：
- 定位：近战高伤害单位
- 生命值：中高
- 攻击力：极高
- 移动速度：慢
- 特殊能力：生命值低于 30% 时，攻击力翻倍

请给出：
1. 详细的属性设计（数值和理由）
2. 与现有单位的对比
3. 可能的平衡性问题
4. 建议的调整方案
```

**步骤 2：配置阶段**
```
基于刚才的设计，请修改 @UnitConfig.cs

要求：
- 添加 BERSERKER 配置
- 保持与现有配置风格一致
- 添加详细注释
```

**步骤 3：逻辑阶段**
```
请实现狂战士的特殊能力：

相关文件：
@Unit.cs - 单位基类
@CombatSystem.cs - 战斗系统

要求：
- 添加 HasBerserkAbility 属性
- 生命值低于 30% 时触发狂暴
- 狂暴时攻击力翻倍
- 添加视觉反馈（如变红）
- 确保 AI 能正确处理这个能力
```

**步骤 4：视觉阶段**
```
请创建狂战士的视觉表现：

参考：@UnitVisuals.cs - 其他单位的视觉

要求：
- Lowpoly 风格
- 使用基础几何体
- 颜色：红色主题
- 能看出是狂战士（如双斧）
```

**步骤 5：测试阶段**
```
请创建测试代码验证狂战士：

测试项：
1. 狂暴技能正确触发（30% 生命值）
2. 攻击力正确翻倍
3. 视觉反馈正常
4. AI 能正确使用
5. 不影响其他单位

请写一个 Editor 测试脚本
```

### 技巧 4：让 AI 自己验证

**❌ 不好的做法**：
```
改完后我自己测试
```

**✅ 好的做法**：
```
请修改代码，然后：
1. 解释你修改了什么
2. 说明可能的影响
3. 给出测试建议
4. 如果可能，写一个自动化测试
```

**示例输出**：
```
我修改了以下文件：
1. UnitConfig.cs - 添加 BERSERKER 配置
2. Unit.cs - 添加狂暴逻辑
3. UnitVisuals.cs - 创建视觉

影响分析：
- 不影响现有单位
- 可能需要调整游戏难度
- 建议降低狂战士的招募成本

测试建议：
1. 测试狂暴触发时机
2. 测试攻击力计算
3. 测试 AI 行为
4. 进行数值平衡测试

我已创建测试脚本：BerserkerTest.cs
```

## 🤖 使用不同的 AI 模型

### Claude（逻辑和架构）

**优势**：
- 擅长写复杂的逻辑代码
- 理解上下文能力强
- 适合架构设计

**使用场景**：
- 设计系统架构
- 实现复杂算法
- 重构代码
- 写文档

**示例提示词**：
```
设计一个技能系统的架构，要求：
- 可扩展（容易添加新技能）
- 解耦（技能不影响核心逻辑）
- 支持 AI 使用
- 有清晰的配置接口

请给出：
1. 架构设计（类图和关系）
2. 核心接口定义
3. 实现示例（2 个技能）
4. 使用说明
```

---

### Gemini（视觉和审美）

**优势**：
- 擅长视觉相关的代码
- 审美判断较好
- 适合 UI 和特效

**使用场景**：
- UI 布局和样式
- 颜色搭配
- 动画效果
- 特效设计

**示例提示词**：
```
设计一个 Lowpoly 风格的骑士角色：

要求：
- 能看出是骑士（盔甲、剑）
- Lowpoly 风格（少量多边形）
- 使用基本几何体组合
- 颜色协调（银色+蓝色）

请给出：
1. 视觉设计方案
2. 几何体组合代码
3. 颜色配置
4. 材质设置
```

---

### GPT（Debug 和分析）

**优势**：
- 擅长找问题
- 分析错误信息
- 给出解决方案

**使用场景**：
- Debug 错误
- 性能分析
- 代码审查
- 问题定位

**示例提示词**：
```
我的游戏在手机上卡顿，请帮我分析：

系统信息：
- 设备：iPhone 12
- 帧率：20-30 FPS
- 目标：60 FPS

当前情况：
- 单位数量：50
- 敌人数量：30
- 地形：512x512

请分析：
1. 可能的性能瓶颈
2. 优化建议（优先级排序）
3. 具体优化代码
4. 预期性能提升
```

## 🎯 实战案例：完整的 AI 协作流程

### 任务：添加弓箭手单位

#### 对话 1：设计阶段

**我**：
```
我想添加一个弓箭手单位，帮我设计：

定位：远程攻击单位
要求：与现有的骑士和战士区分开

请给出：
1. 详细的属性设计
2. 与其他单位的对比
3. 平衡性考虑
```

**AI**：
```
弓箭手属性设计：
- 生命值：60（低，因为是远程）
- 攻击力：15（中等）
- 攻击范围：10（远）
- 攻击速度：1.5s/次（慢）
- 移动速度：5（中等）
- 特殊：穿透攻击（能射中一条线上的敌人）

对比：
- 骑士：高血量近战坦克
- 战士：中等血量近战输出
- 弓箭手：低血量远程输出

平衡性：
- 弓箭手脆皮，需要保护
- 远程优势但装填慢
- 穿透效果对密集敌人有效
```

#### 对话 2：实现阶段

**我**：
```
基于这个设计，请实现弓箭手：

相关文件：
@UnitConfig.cs
@Unit.cs
@UnitVisuals.cs

要求：
1. 添加配置
2. 实现远程攻击逻辑
3. 创建视觉（带弓和箭）
4. 实现穿透效果
```

**AI**：
```
修改了以下文件：

1. UnitConfig.cs - 添加 ARCHER 配置
2. Unit.cs - 添加 IsRanged 属性
3. RangedCombat.cs - 新建远程战斗系统
4. UnitVisuals.cs - 创建弓箭手视觉
5. Projectile.cs - 新建箭矢类

关键改动：
- 远程攻击使用箭矢对象
- 箭矢有碰撞检测
- 穿透逻辑：击中后继续飞行，伤害衰减

测试建议：
1. 测试射程和精度
2. 测试穿透效果
3. 测试 AI 使用弓箭手
```

#### 对话 3：调试阶段

**我**：
```
箭矢总是射偏目标，请帮我调试：

相关代码：
@RangedCombat.cs
@Projectile.cs

现象：
- 箭矢总是从头顶飞过
- 近距离射击会打空

请分析原因并修复
```

**AI**：
```
问题分析：
1. 箭矢生成位置太高（在头顶而不是胸前）
2. 没有考虑目标高度（只瞄准目标的脚部）
3. 弹道是直线，应该有抛物线

修复方案：
1. 调整箭矢生成位置到胸前
2. 计算目标中心点（头部+脚部的一半）
3. 添加简单的重力模拟

修改了：
- RangedCombat.cs 第 45 行：调整生成位置
- RangedCombat.cs 第 52 行：计算目标中心
- Projectile.cs：添加重力模拟

测试建议：
- 测试不同距离的命中率
- 测试对不同高度单位的射击
```

#### 对话 4：优化阶段

**我**：
```
弓箭手 AI 还不够智能：

问题：
- 不保持安全距离
- 不会集中火力
- 不会优先攻击高价值目标

请改进 AI：
@AIController.cs
@TargetSelector.cs
```

**AI**：
```
添加了以下智能行为：

1. 安全距离保持：
   - 敌人太近时后撤
   - 保持 5-8 单位距离

2. 集中火力：
   - 优先攻击已被攻击的目标
   - 减少溢出伤害

3. 目标优先级：
   - 低血量 > 高血量
   - 远程 > 近战 > 坦克
   - 距离近 > 距离远

修改了：
- AIController.cs - 添加射击位置计算
- TargetSelector.cs - 添加优先级评分
- UnitAI.cs - 整合新行为

性能影响：
- 增加了约 5% 的 CPU 开销
- 建议每帧只更新部分 AI
```

## 💪 最佳实践总结

### DO ✅

1. **给 AI 充分的上下文**
   - 项目结构
   - 相关代码
   - 具体需求
   - 预期结果

2. **分步骤处理复杂任务**
   - 设计 → 实现 → 测试 → 优化
   - 每步都要明确目标

3. **让 AI 验证自己的代码**
   - 解释改动
   - 分析影响
   - 写测试代码

4. **建立项目记忆**
   - README.md
   - ARCHITECTURE.md
   - TODO.md
   - CHANGELOG.md

5. **使用合适的 AI 模型**
   - Claude 写逻辑
   - Gemini 做视觉
   - GPT 做 Debug

### DON'T ❌

1. **不要给模糊的指令**
   - ❌ "优化这个"
   - ✅ "将性能提升 30%，目标是 60 FPS"

2. **不要跳过验证**
   - ❌ "改完直接用"
   - ✅ "改完并写测试验证"

3. **不要忽视影响范围**
   - ❌ "只改这个函数"
   - ✅ "分析影响范围并修改相关代码"

4. **不让 AI 记住项目**
   - ❌ 每次都从头解释
   - ✅ 用文档建立记忆
